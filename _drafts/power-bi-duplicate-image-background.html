In Power BI when we use a background image on our pages, our Power BI file can become a bit more heavier.

Keep an eye on the image you use in your background.

It’s simple, if your image is heavy, then your report will be too. No need to use a very high definition, if it exceed the one that your Power BI Report use.

Power BI store each background

No matter if you reuse the same image on each page, when you add an image in the canvas background settings, Power BI will store it in your .PBIX file.

You can do this experimentation by duplicate your power bi, rename the .pbix extension into a .zip extension, and extract it. If you go to the Report > StaticResources > RegisteredResources, all the wallpaper used will be there.

Ways to reduce the size

But you can change this behaviour! And how?

Duplicate your page

When you duplicate a page where an background image exist, Power BI will reuse it. When your report has 10 pages with a 500Kb background, it starts to be interesting.

Use a base 64 image into a theme

This is the option I use. It’s not too complicated (a bit more than the previous one), and it will be applied to all your pages.

I’m that too lazy, but somebody already explained it very well! https://pbi-guy.com/2021/11/05/power-bi-theme-with-background-image/

And use this link to transform your image into base 64 https://onlinejpgtools.com/convert-jpg-to-base64

{
    "name": "MyTheme",
    "visualStyles": {
        "page": {
            "*": {
                "background": [
                    {
                        "image": {
                            "name": "LazyDAX_Background",
                            "scaling": "Fit",
                            "url": "data:image/png;base64,XXXXReplaceByTheGeneratedCodeXXXX"
                        },
                        "transparency": 0
                    }
                ]
            }
        }
    }
